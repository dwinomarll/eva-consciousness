# MCP Server Configs

Each file defines one MCP server in two formats:
- `opencode` key → written to `opencode.json` (opencode format)
- `claude` key → written to `.mcp.json` (Claude Code format)

The spawn CLI reads these at spawn time and writes both config files
with only the servers you selected.

## Available Servers

| Server | File | Requires |
|---|---|---|
| Filesystem | filesystem.json | Nothing — reads/writes `.` |
| Memory | memory.json | Nothing — persists to `~/eva-workspace/memory/memory.json` |
| Sequential Thinking | sequentialthinking.json | Nothing |
| Gmail | gmail.json | One-time Google OAuth through the package auth flow |
| Google Calendar | gcal.json | `GOOGLE_OAUTH_CREDENTIALS` |
| Slack | slack.json | `SLACK_BOT_TOKEN` + `SLACK_TEAM_ID` |
| Notion | notion.json | `NOTION_TOKEN` in .env |
| Omi | omi.json | `OMI_API_KEY` in .env + Docker |

## Eva Tether

These entries are the external tool belt that spawned playgrounds can use.
The Eva tether itself is generated by the language template:

- Node stdio: `npm run mcp`
- Node Streamable HTTP: `EVA_AUTH_TOKEN=<secret> npm run mcp:http`
- Python stdio: `python eva_mcp.py` or `uv run eva_mcp.py`

Use the HTTP tether for MAAT/FIRE or any other off-host MCP client. Point the
client at `http://<host>:8787/mcp` and store the bearer token in the caller's
secret system by name, not in workflow text.
